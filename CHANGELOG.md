# Changelog

DID/VC学習・実装プロジェクトの変更履歴

## [Unreleased] - 2025-01-26

### QRコード文字化け修正
- **Base64エンコーディング対応**: 日本語を含むVPデータのQRコード処理で文字化けエラーを解消
- **UTF-8安全化**: `btoa(unescape(encodeURIComponent()))`でエンコード、`decodeURIComponent(escape(atob()))`でデコード
- **VP検証改善**: QRコード読み取り・JSON解析・VP検証の全工程でBase64対応

## 2025-09-21

### ダークテーマ・ライトテーマ両対応システム実装
- **Bootstrap 5標準対応**: CSS Custom Propertiesを使用したdata-bs-theme切り替えシステム
- **テーマ統一**: 全画面でダークテーマ・ライトテーマ両対応（手動切り替え）
- **VCカード専用スタイル**: デジタル身分証明書カードの独立した色調整
- **SweetAlert2対応**: モーダル表示のテーマ別スタイル調整
- **CSS最適化**: Bootstrap変数活用で保守性向上、!important使用削減
- **シナリオボタン専用クラス**: CSS詳細度競合解決、認証後の選択状態維持
- **フォント統一**: 日本語システムフォント適用、ボタン丸み調整（50px）

## 2025-09-20

### ライブラリアーキテクチャ整備・入力バリデーション・JSDoc完全対応
- **ライブラリ分離**: 共通機能を3つのライブラリに分割（cryptoUtils.js, didRegistry.js, vcManager.js）
- **モジュール移行**: 各画面関数をライブラリ関数使用に統一、将来のブロックチェーン移行対応
- **入力バリデーション**: VC申請フォームにHTML5標準バリデーション・Bootstrap視覚的フィードバック実装
- **文字数制限**: 氏名20文字・住所60文字のmaxlength制限追加
- **JSDoc完全対応**: 全37関数に統一フォーマットの詳細ドキュメント追加
- **QRコードサイズ最適化**: VP生成時の軽量化でサイズ制限エラー解消
- **Unicode対応**: 日本語文字のBase64エンコード問題修正
- **ドキュメント更新**: CLAUDE.md・README.mdのディレクトリ構成図を最新状態に反映

## 2025-09-19

### UI/UX改善とフロー最適化
- **SweetAlert2導入**: 認証結果表示を統一されたモーダルに変更
- **文言統一**: 「本人確認認証」→「本人確認」に簡略化
- **認証後フロー改善**: シナリオ選択を維持して連続テスト可能
- **表示最適化**: 認証日時削除、発行機関配置変更、チェック形式統一
- **ボタン改善**: 全幅表示、ラベル簡略化、スタイル統一
- **デバッグログ整理**: 重要でないログをコメントアウト

## 2025-09-17

### モジュール化・ビルド環境整備
- **CDNからnpmパッケージへ移行**: Bootstrap, qrcode-generator, html5-qrcodeをnpm管理
- **FOUC問題解決**: CSSインポートを`style.css`に統一し、スタイル適用の瞬断を解消
- **デプロイ環境対応**: Viteの`base: './'`設定でサブディレクトリ配置に対応
- **ディレクトリ構造統一**: 全HTMLファイル対応のJavaScriptファイル作成
- **ビルド最適化**: 相対パス対応、Tree-shaking、コード分割

## 2025-09-16

### リアカメラQR読み取りと統一ローディング実装
- **QRスキャン機能**: html5-qrcode導入、リアカメラ自動起動
- **統一ローディングUI**: 共通ローディングライブラリ実装
- **UI/UX改善**: ボタンデザイン統一、プログレッシブディスクロージャー採用
- **技術改善**: ES6モジュール対応、QRコード生成最適化
