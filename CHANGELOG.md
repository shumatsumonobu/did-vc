# Changelog

DID/VC プロジェクトの変更履歴


## 2025-01-26

### QRコード文字化け修正

日本語を含むVPデータのQRコード処理で発生していた文字化けを解消。

- Base64エンコーディング対応 — `btoa(unescape(encodeURIComponent()))` / `decodeURIComponent(escape(atob()))` でUTF-8安全化
- VP検証改善 — QRコード読み取り・JSON解析・VP検証の全工程でBase64対応


## 2025-09-21

### ダークテーマ / ライトテーマ両対応

Bootstrap 5の `data-bs-theme` 切り替えによるテーマシステムを実装。

- CSS Custom Propertiesベースの全画面テーマ対応（手動切り替え）
- VCカード・SweetAlert2モーダルの個別スタイル調整
- Bootstrap変数活用で `!important` 使用を削減、保守性向上
- シナリオボタン専用クラス導入 — CSS詳細度競合を解決し、認証後の選択状態を維持
- 日本語システムフォント統一、ボタン丸み調整（50px）


## 2025-09-20

### ライブラリ分離とバリデーション整備

共通機能を3ライブラリに分割し、入力バリデーションとJSDocを全面導入。

| 対象 | 内容 |
|:-----|:-----|
| ライブラリ分離 | cryptoUtils.js / didRegistry.js / vcManager.js に分割 |
| モジュール統一 | 各画面関数をライブラリ呼び出しに移行、将来のブロックチェーン対応を見据えた設計 |
| 入力バリデーション | HTML5標準 + Bootstrap視覚フィードバック、氏名20字・住所60字制限 |
| JSDoc | 全37関数に統一フォーマットのドキュメント追加 |
| QRコード | VP生成時の軽量化でサイズ制限エラー解消、日本語Base64対応 |


## 2025-09-19

### UI/UX改善とフロー最適化

認証結果表示をSweetAlert2モーダルに統一し、操作フローを簡潔化。

- 文言統一 — 「本人確認認証」→「本人確認」
- 認証後フロー改善 — シナリオ選択を維持し連続テスト可能に
- 表示最適化 — 認証日時削除、発行機関配置変更、チェック形式統一
- ボタン — 全幅表示、ラベル簡略化、スタイル統一


## 2025-09-17

### モジュール化とビルド環境整備

CDN依存からnpmパッケージ管理に移行し、Viteベースのビルド環境を構築。

- Bootstrap / qrcode-generator / html5-qrcode をnpm管理に移行
- FOUC解消 — CSSインポートを `style.css` に統一
- Vite `base: './'` でサブディレクトリ配置対応
- Tree-shaking・コード分割による最適化


## 2025-09-16

### リアカメラQR読み取りと統一ローディング

html5-qrcodeによるリアカメラ自動起動のQRスキャン機能と、共通ローディングUIを実装。

- ボタンデザイン統一、プログレッシブディスクロージャー採用
- ES6モジュール対応、QRコード生成最適化
